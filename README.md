# Target-Skills
An interactive web application designed to bridge the gap between job seekers and their dream companies. Users can search for any company to discover its required skill stack, then instantly generate a personalized, step-by-step learning roadmap for any specific skill.

## 📌 Features

* **Company Skill Stack Discovery:** Search for top-tier companies and instantly pull their current tech stack, core competencies, and framework requirements.
* **AI-Powered Personalized Roadmaps:** Choose any specific skill and generate an interactive, step-by-step milestone curriculum custom-tailored to your current level.
* **Curated Repositories & Study Guides:** Access handpicked, lightweight open-source repositories and study guides dynamically paired with each roadmap phase.
* **Visual Capstone Validation:** Validate your knowledge by completing specific terminal project targets recommended for each track.

# ⚙️ Installation

### 1. Clone the repository

```bash
git clone https://github.com/Bobbykumar5158/Target-Skills.git
```

### 2. Move into the project folder

```bash
cd Target-Skills
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Create a `.env` file

```env
GEMINI_API_KEY=your_api_key
```

### 5. Run the application

```bash
python app.py
```

Open your browser and visit:

```
http://127.0.0.1:8080
```

## 📝 Project Reflection

Building **Target-Skills** was an intensive exercise in bridging unstructured generative AI outputs with rigid frontend rendering. 

### What was the hardest part of building this?
The most difficult challenge was structural state synchronization—specifically, parsing LLM JSON responses consistently. Managing dynamic data schemas generated by the model and mapping them directly to our Flask/Jinja templates often caused rendering errors when the model occasionally changed key names or omitted nested properties (such as steps or subtopics). Ensuring robust error boundaries and fallback validation logic in `DB.py` and `genai.py` required extensive debugging.

### What would you improve with more time?
With additional time, I would transition the application from using simple static JSON files (`user.json` and `roles.json`) to a relational PostgreSQL database schema. This would allow for structured user progress tracking, saving multiple custom-generated roadmaps per user profile, and performing faster data queries than reading/writing text files on disk.

### What skill from this program do you feel genuinely confident in?
I now feel genuinely confident in frontend layout architecture and UX design hierarchy. Translating complex JSON trees into a clean, layered, ice-blue visual timeline using CSS variables, custom grid systems, and modular components (like the Core Capstone cards) taught me how to visually structure heavy information so that it remains digestible and intuitive for the end-user.
